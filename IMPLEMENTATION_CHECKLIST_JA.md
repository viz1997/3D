## 快速故障排除

### 问题1: Meta Description在SERP中显示不完整

**症状**: Google搜索结果中的描述被截断

**解决方案**:
```
□ 检查字数: 应该是155-160字
  当前: ___字 (数一下)
□ 检查是否有特殊字符破坏
  删除多余的特殊符号，保留必要的
□ 在Google SERP preview中再次检查
□ 清除Google缓存:
  1. GSC中找到网址
  2. 点击3个点菜单
  3. 点击 "清除缓存"
```

---

### 问题2: Title在SERP中显示与预期不同

**症状**: Google搜索结果显示的Title与设置的不同

**可能原因与解决**:
```
□ Google改写了Title (太常见)
  原因: Google认为你的Title不够吸引人
  解决:
    1. 检查当前Title是否足够独特
    2. 添加更多情感触发词
    3. 等待2-4周，Google可能会停止改写

□ 网页有多个<title>标签
  解决:
    1. 检查页面源代码 (Ctrl+U)
    2. 确保只有1个<title>标签
    3. 删除多余的title标签

□ Meta Title太长 (超过60字)
  解决:
    1. 字数应该是45-55字
    2. 删除不必要的词

□ Title被缓存了 (较新的改动)
  解决:
    1. 在GSC中提交URL重新爬取
    2. 等待24-72小时
```

---

### 问题3: Schema验证失败

**症状**: Schema validator显示错误

**常见错误与解决**:
```
错误: "Missing 'price' field"
原因: Schema中缺少price字段
解决: 确保有以下代码:
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "JPY"
  }

错误: "Invalid schema type"
原因: "@type" 值拼写错误
解决: 确保是 "SoftwareApplication" 而非其他

错误: "URL mismatch"
原因: Schema中的URL与实际页面URL不符
解决: 在 "url" 字段中填入正确的完整URL
```
